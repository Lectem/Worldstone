17 ## Status of the project
18 
19 DC6 and DCC (sprites) decoders and renderer are now working.
20 PL2 file (palette shifts) generation is working and gives the same results as if created by the game. Optional fixes are available for more accurate palette shifts.
21 
22 The [viewer app](https://github.com/Lectem/Worldstone/tree/master/source/tools/RendererApp) can be used to browse Mpq files and display the following:
23 * .dc6 sprites
24 * .dcc sprites
25 * .dat and .pl2 palettes and palshifts
26 * .cof animation files information (no sprite display yet)

30 ## Building
31 
32 ### Prerequisites
33 * CMake 3.6 or higher
34 * A C++14 compiler
35  - GCC 6.0 - tested (5.0 should work)
36  - clang 3.8 with libc++ or libstdc++-5(only 6 is tested) - tested
37  - MSVC 14 (Visual 2015) - tested
38 * Bzip2 for Stormlib(libbz2-dev on debian/ubuntu)
39 
40 ### Copy-paste from command line (in the project directory):
41 
42  git submodule update --init --recursive
43  mkdir build && cd build
44  cmake ..
45  cmake --build .
46 
47 ### Configuration
48 
49 You can tweak the configuration using the following CMake variables (I suggest using ccmake or cmake-gui)
50 * ENABLE_LTO: Use link time optimization on non debug builds if available
51 * ENABLE_WARNINGS_SETTINGS: Set to false to remove warning flags added by the project
52 * BUILD_TESTING: Disable all testing features
53 * WS_BUILD_TESTS: Disable Worldstone tests
54 * WS_USE_DOXYGEN: Should we (try to) add a doc target
55 * WS_WITH_BGFX: Build targets that require bgfx (Anything doing rendering)
56 
57 ### On Windows
58 
59 - Required shared libraries should automatically be copied to the executable directory by CMake.
